# 架构

PoPOpt 以单个 ASI 形式发布，但其源代码分为一个小型核心和独立的特性模块。

## 启动阶段

### `DllMain` 中的同步基础初始化

游戏可以在 ASI 加载器返回后立即查询和缓存 `LNG_Language`。因此，只有启动关键的语言/语音基础部分会同步运行：

1. 解析 ASI/游戏目录和固定输出路径。
2. 读取 `Debug.LogLevel` 并初始化早期日志记录。
3. 检测可执行文件风格。
4. 解析请求的语音语言和后备包。
5. 注册并安装注册表和语音文件 IAT 钩子。
6. 应用静态语音选择器组，或准备 SteamStub 运行时补丁。

在此阶段不执行任何 D3D9、DirectInput、窗口操作、DPI、CPU 亲和性或报告工作。

### 工作线程初始化

工作线程注册完整的模式（schema）、验证配置、初始化其余模块、安装它们的钩子、应用非基础补丁、启动后台工作线程并编写报告。

## 核心

- `Core`：应用程序上下文、Unicode 路径、INI 生成/验证、日志记录器、报告编写器。
- `HookManager`：一个由所有模块共享的、具有所有者感知能力的 IAT 安装路径。
- `PeUtils`：EXE 风格检测、RVA 验证、掩码匹配、事务性内存补丁。
- `Module`：通用生命周期回调。

## 模块

- `SettingsRegistry`：文本语言和游戏注册表值虚拟化。
- `Voice`：包检测、后备、四选择器语音补丁、SteamStub 运行时处理。
- `Display`：窗口模式和无边框窗口行为。
- `DPI`：进程 DPI 感知。
- `CPU`：亲和性限制。
- `Input`：DirectInput Win 键行为和焦点丢失鼠标释放。
- `Graphics`：图形配置验证和报告。
- `TextureLoader`：TPF/ZIP 解析、包优先级、D3D9 钩子、替换生命周期。
- `Diagnostics`：兼容性报告聚合。

## 模块隔离规则

1. 模块依赖核心（core），而不相互依赖。
2. 共享状态通过 `AppContext`、配置、事件和钩子管理器暴露。
3. 每个模块必须能够容忍重复的生命周期调用；启动关键模块使用显式的互锁守卫。
4. 禁用的模块不得安装钩子或加载可选的库。
5. 报告贡献到一份兼容性报告，而不是单独的文件。

## 纹理匹配

集成加载器使用经游戏内测试控制器提示包验证的模式：

```text
top-compact-complement
```

它计算 mip 级别 0 的有效行（排除填充字节）的 CRC32，然后对结果取补码。匹配仅基于哈希值；源纹理和替换纹理的尺寸不必匹配。
